# 大转盘 — 需求规格文档

> 版本：v1.0  
> 更新日期：2024-06-15

---

## 1. 项目概述

### 1.1 项目名称
**大转盘**（Lucky Spin Wheel）

### 1.2 项目目标
为日常决策困难（吃什么、喝什么）提供一个简单有趣的随机选择工具。用户可以在手机或电脑上使用，无需安装，打开浏览器即可。

### 1.3 目标用户
- 不懂电脑的普通用户（"小白"）
- 只需要最基本的功能，界面越简单越好
- 主要在手机上使用，偶尔在电脑上查看

---

## 2. 功能需求

### 2.1 转盘核心功能

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F1 | 转盘绘制 | 用 Canvas 绘制彩色扇区转盘，每个扇区显示选项文字 | P0 |
| F2 | 旋转动画 | 点击按钮后转盘旋转，带缓出减速效果，最终随机停止 | P0 |
| F3 | 指针指示 | 顶部固定指针，指示中选结果 | P0 |
| F4 | 结果弹窗 | 转盘停止后弹出结果提示 | P1 |

### 2.2 分类管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F5 | 分类切换 | 顶部标签切换"🍜 吃的"和"🧋 喝的"，各自独立的转盘和选项 | P0 |
| F6 | 默认选项 | 首次使用时每个分类预置 6-8 个常见选项 | P1 |

### 2.3 选项编辑

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F7 | 添加选项 | 输入框 + 添加按钮，可新增选项 | P0 |
| F8 | 删除选项 | 每个选项旁有删除按钮（最少保留 2 个选项） | P0 |
| F9 | 修改选项 | 点击选项文字可编辑（双击或长按） | P1 |

### 2.4 历史记录

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F10 | 自动记录 | 每次转出结果自动保存，记录日期和选项名 | P0 |
| F11 | 历史列表 | 可查看历史记录列表，按时间倒序排列 | P0 |
| F12 | 上传照片 | 对某条记录可上传/拍摄照片，留作纪念 | P2 |
| F13 | 删除记录 | 可删除某条历史记录 | P2 |

### 2.5 数据管理

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| F14 | 数据持久化 | 所有数据存 localStorage，刷新不丢失 | P0 |
| F15 | 数据重置 | 提供"恢复默认"按钮，清空所有数据并恢复默认选项 | P2 |

---

## 3. 非功能需求

### 3.1 跨平台
- 手机（iOS Safari / Android Chrome）可正常使用
- 电脑（Chrome / Edge / Firefox）可正常使用
- 响应式布局，自动适配屏幕尺寸

### 3.2 性能
- 转盘动画流畅（60fps）
- 页面加载 < 1 秒
- 图片压缩后单张 < 100KB

### 3.3 可用性
- 零学习成本，打开即用
- 按钮足够大，适合手指点击（最小 44px）
- 文字清晰可读（最小 14px）

### 3.4 离线
- 不需要网络连接
- 所有功能离线可用

---

## 4. 不做的事项（明确排除）

- ❌ 不接入后端/服务器
- ❌ 不需要用户注册/登录
- ❌ 不支持多设备同步
- ❌ 不需要音效
- ❌ 不需要分享功能
- ❌ 不需要多语言

---

## 5. 用户场景

### 场景 1：决定午餐
> 小王中午不知道吃什么，打开大转盘，切换到"吃的"，点一下"开始"按钮，转盘转了几圈后停在"麻辣烫"，小王满意地去吃了。吃完回来上传了一张麻辣烫的照片。

### 场景 2：编辑选项
> 小王最近不想吃寿司了，打开选项列表，把"寿司"删掉，加了一个"螺蛳粉"。刷新页面后选项还在。

### 场景 3：回顾记录
> 小王想看看这周都吃了什么，打开历史记录，看到一周的选择记录，有的还附了照片。
